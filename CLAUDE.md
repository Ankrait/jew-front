## Обзор проекта

JewelBot - это приложение-чатбот для ювелирного интернет-магазина, созданное на React, TypeScript и Vite. Приложение предоставляет два основных представления:
- **Клиентский вид**: Интерфейс для покупателей с демонстрацией ювелирных изделий и AI-ассистентом
- **Панель для бизнеса**: Админ-панель для управления диалогами, просмотра аналитики и настройки бота

## Технологический стек

- **Фронтенд**: React 18 с TypeScript
- **Сборщик**: Vite
- **Стили**: Tailwind CSS
- **Управление состоянием**: Redux Toolkit (подключен, но еще не реализован)
- **Иконки**: Lucide React
- **Обмен данными в реальном времени**: Socket.io-client (подключен, но не реализован)
- **Бэкенд**: Supabase (подключен, но не реализован)

## Архитектурные особенности

### Текущее состояние реализации
- Приложение является прототипом только с фронтендом и моковыми данными
- Компонент ChatBot использует простое сопоставление шаблонов для ответов
- Админ-панель отображает статичную аналитику
- Пока нет интеграции с бэкендом (Supabase и Socket.io установлены как зависимости, но не используются)

### Ключевые компоненты

**ClientView** (`src/components/ClientView.tsx`)
- Отображает каталог ювелирных изделий по категориям (кольца, ожерелья, серьги)
- Плавающая кнопка чата для открытия виджета ChatBot
- Адаптивный дизайн с использованием градиентов на Tailwind

**ChatBot** (`src/components/ChatBot.tsx`)
- Простой интерфейс чата с историей сообщений
- Ответы бота на основе паттернов для типичных запросов о ювелирных изделиях
- Автопрокрутка к последним сообщениям
- Сейчас без интеграции с реальным AI/бэкендом

**AdminView** (`src/components/AdminView.tsx`)
- Три вкладки: Диалоги, Аналитика, Настройки
- Управление сессиями чата (моковое)
- Дашборд с метриками по каналам
- Анализ трендов товаров (высокий/низкий спрос)
- Настройки бота (нефункциональные)

### Подход к стилизации
- Используются utility-классы Tailwind CSS
- Единая цветовая схема: purple-600 для основных действий, slate для нейтральных элементов
- Адаптивный дизайн с мобильным подходом
- Градиенты и тени для создания глубины

## Рекомендации по разработке

1. **Язык интерфейса**: Приложение в основном на русском языке
2. **Цветовая схема**: Соблюдать консистентность с purple-600 как основным цветом
3. **Паттерн компонентов**: Функциональные компоненты с хуками
4. **Типизация**: Все компоненты должны иметь TypeScript интерфейсы
5. **Адаптивность**: Использовать адаптивные утилиты Tailwind (md:, lg: брейкпоинты)
